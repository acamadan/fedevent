Google Places API Troubleshooting

Resolving "API key not valid" Error

Current Error Details

You're encountering the following error:

API Error: API key not valid. Please pass a valid API key.

{
  "error": {
    "code": 400,
    "message": "API key not valid. Please pass a valid API key.",
    "status": "INVALID_ARGUMENT",
    "details": [
      {
        "@type": "type.googleapis.com/google.rpc.ErrorInfo",
        "reason": "API_KEY_INVALID",
        "domain": "googleapis.com",
        "metadata": {
          "service": "places.googleapis.com"
        }
      }
    ]
  }
}

This error indicates that while your API key is loaded correctly in the application, it's not valid for the Google Places API (New). Let's systematically troubleshoot this issue.

Immediate Solution

Based on your error, here's what you need to do:

  1. Wait for API key propagation - Google mentions it can take up to 5 minutes for settings to take effect
  2. Verify Google Cloud Console configuration - Ensure all settings are correctly applied
  3. Clear browser cache - For localhost:5050 to ensure you're not loading cached validation results

Google Cloud Console Configuration Checklist

1. Enable Required APIs

Ensure these APIs are enabled in your Google Cloud Console project:

  • ✅ Maps JavaScript API - Required for map display features
  • ✅ Places API - The original Places API
  • ✅ Places API (New) - The new version you're using

How to check: Go to Google Cloud Console > APIs & Services > Library, search for each API and ensure it shows as "API enabled"

2. API Key Restrictions

Verify your API key restrictions are properly configured:

  • ✅ Application restrictions - Set to "HTTP referrers (websites)"
  • ✅ HTTP referrers - Must include http://localhost:5050/*
  • ✅ API restrictions - Set to "Restrict key" with all three APIs checked

How to check: Go to Google Cloud Console > APIs & Services > Credentials > Click on your API key > Verify restrictions section

3. Billing Configuration

Confirm your project has billing enabled:

  • ✅ Billing account - Linked to your project
  • ✅ Billing status - Active and in good standing

How to check: Go to Google Cloud Console > Billing > Verify your project is linked to an active billing account

4. API Key Verification

Double-check your API key is correct:

  • ✅ API key in .env file - Matches exactly with Google Cloud Console
  • ✅ No extra spaces - Before or after the key
  • ✅ Correct key type - Server key (not iOS/Android key)

How to check: Compare the key in your .env file with the one in Google Cloud Console

Advanced Troubleshooting Steps

1. Create an Unrestricted Test Key

For diagnostic purposes only:

  • Go to Google Cloud Console > APIs & Services > Credentials
  • Click "+ CREATE CREDENTIALS" > "API key"
  • DO NOT add any restrictions to this key
  • Copy the key and temporarily replace your current key in the .env file
  • Test your application
  • IMPORTANT: Delete or restrict this key immediately after testing

2. Check for Multiple API Keys

Ensure you're using the correct API key:

  • Verify you don't have multiple API keys in Google Cloud Console
  • Ensure you're using the key with the correct restrictions
  • Check if there are any expired or deactivated keys

3. Verify Project ID

Ensure your API key is associated with the correct project:

  • Check that you're working in the correct Google Cloud project
  • Verify the project has the correct name and ID
  • Ensure all APIs are enabled in the same project where your key was created
